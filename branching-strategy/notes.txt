🌿 Git Branching Strategy — Quick Notes
🔹 What is a Branch?

A branch is a safe copy of the project where I can work without breaking main.

👉 Work → test → merge later

🎯 Why I Use Branches
Working on main is risky ❌
No safe way to test ❌

With branches:

✔ Safe experimentation
✔ No impact on main
✔ Can fix mistakes before merging

🏗️ Types of Branches
🔵 main
Final working version
Always stable
Never code directly

👉 Used for production

🟢 develop
Main working branch
All features come here
Not fully stable

👉 Work-in-progress version

🟣 feature/*
One feature per branch
Created from develop
Merged back to develop

👉 “1 branch = 1 feature”

🟡 release/*
Prepare for production
Only bug fixes allowed
No new features

👉 Final testing stage

🔴 hotfix/*
Fix production issues
Created from main
Merged into main + develop

👉 Emergency fixes

## 🔄 Hands-on Practice I Performed

Created develop from main
Created feature branches
Merged features into develop
Created release branch
Merged into main
Created hotfix and merged

🧠 What I Learned

✔ Never work on main directly
✔ Always use branches
✔ Keep features small
✔ Delete branches after use

🏁 Final Understanding

Branching helps me:

Work safely
Stay organized
Avoid breaking code

👉 Feels like:
“I build features separately and merge only when ready”
